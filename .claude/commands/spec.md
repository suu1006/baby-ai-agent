# 기능 명세 에이전트

새로운 기능을 개발하기 전에 구체적인 명세를 작성합니다. 명세가 명확할수록 AI가 더 정확한 코드와 테스트를 생성할 수 있습니다.

## 실행 절차

### 1단계: 기능 파악

사용자에게 아래 질문을 순서대로 하세요. 답변이 충분하면 다음 질문으로 넘어가세요.

1. **무엇을 만드나요?** — 기능 이름과 한 줄 설명
2. **어떤 화면/컴포넌트에서 동작하나요?** — 위치와 맥락
3. **사용자는 어떤 행동을 하나요?** — 입력, 탭, 스크롤 등
4. **정상 동작 결과는 무엇인가요?** — 성공 시 화면에 보이는 것
5. **실패하거나 예외 상황은 무엇인가요?** — 에러, 빈 상태, 네트워크 오류 등

### 2단계: 명세 문서 작성

수집한 답변을 바탕으로 `docs/spec-{기능명}.md` 파일을 생성하세요.

파일 형식:

```markdown
# 기능명

## 개요
한 줄 설명

## 대상 화면
- 파일 경로 또는 컴포넌트명

## 사용자 시나리오

### 정상 흐름
1. 사용자가 ~을 한다
2. 시스템이 ~을 보여준다
3. ...

### 예외 흐름
- 상황: 결과
- 상황: 결과

## 입력/출력
- 입력: (props, 사용자 입력, API 응답 등)
- 출력: (화면 변화, 상태 변경, API 호출 등)

## 제외 범위
- 이번 구현에서 다루지 않는 것
```

### 3단계: 완료 보고

---
**[명세 작성 완료]**

- 저장 위치: `docs/spec-{기능명}.md`
- 다음 단계: `/test`를 실행하면 이 명세를 기반으로 테스트 코드를 생성합니다.

---

## 주의사항
- 모호한 답변은 구체적으로 다시 질문하세요 ("버튼을 누른다" → "어떤 버튼인가요? 탭 어디에 있나요?")
- 명세는 짧고 명확하게 — 구현 방법이 아닌 동작을 기술하세요
- `docs/` 폴더가 없으면 생성하세요
