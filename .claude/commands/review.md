# 코드 리뷰 에이전트

변경된 코드를 GitHub 푸시 전에 검토합니다. 문제가 있으면 사용자에게 보고하고, 수정 여부를 결정하게 합니다.

## 실행 절차

### 1단계: 변경사항 수집

```bash
git diff
git diff --cached
git status
```

변경사항이 없으면 "검토할 변경사항이 없습니다"라고 보고하고 종료하세요.

### 2단계: 코드 리뷰 수행

수집한 변경사항을 아래 항목 기준으로 검토하세요.

**검토 항목:**

1. **보안**
   - 하드코딩된 API 키, 비밀번호, 토큰
   - `.env`에 있어야 할 값이 코드에 직접 포함된 경우
   - 사용자 입력값 미검증

2. **로직 오류**
   - 명백한 버그 (null 참조, 잘못된 조건문, 무한루프 가능성)
   - 비동기 처리 누락 (async/await 없이 Promise 사용)
   - 상태 업데이트 오류 (React 상태 직접 변경 등)

3. **타입 안전성**
   - TypeScript `any` 남용
   - 타입 단언(`as`) 무분별한 사용
   - 누락된 null 체크

4. **코드 품질**
   - 사용되지 않는 import, 변수
   - console.log 디버그 코드 잔류
   - 중복 코드

### 3단계: 결과 보고

아래 형식으로 보고하세요:

---
**[코드 리뷰 결과]**

**검토 파일:**
- 파일 목록

**발견된 문제:**

🔴 **심각** (보안·데이터 손실 위험)
- 없음 또는 구체적 문제와 위치

🟡 **주의** (버그 가능성·타입 오류)
- 없음 또는 구체적 문제와 위치

🔵 **참고** (코드 품질·스타일)
- 없음 또는 구체적 문제와 위치

**종합 의견:**
한 줄 요약

---
심각 문제가 있으면 수정을 권고하세요. 주의·참고만 있으면 사용자가 선택하게 하세요.

수정 후 재검토가 필요한 경우: "수정 완료 후 `/review`를 다시 실행하거나 `/push`로 진행하세요."
문제가 없는 경우: "`/push`로 푸시를 진행하세요."

## 주의사항
- 리뷰는 제안일 뿐, 최종 결정은 사용자가 합니다
- 코드를 직접 수정하지 않습니다 (사용자가 요청할 때만 수정)
- 심각 문제가 아니면 `/push` 진행을 막지 않습니다
